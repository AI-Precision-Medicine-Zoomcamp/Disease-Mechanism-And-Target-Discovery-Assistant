# Disease Mechanism & Target Discovery Assistant

An AI powered assistant for helping researchers explore disease mechanisms, biological pathways, gene networks, and potential therapeutic targets from public biomedical knowledge sources.

The initial focus is disease mechanism understanding and target discovery, not clinical decision-making or drug recommendations. The system combines structured biomedical knowledge (e.g., Open Targets-style data) with retrieval-augmented generation (RAG) to support exploratory research.

## Core Idea

Biomedical knowledge about diseases is distributed across multiple databases and is often complex to interpret in isolation. A researcher trying to understand a disease such as lung cancer may need to manually explore multiple layers of information:

- gene–disease associations
- pathway enrichment data
- protein interactions
- target evidence scoring
- mechanistic descriptions

For example, a typical research question might be:

```text
Which genes and pathways are driving NSCLC progression?
```

The goal of this project is not to replace biomedical databases. Instead, it creates a lightweight discovery assistant that can:

normalize disease and target-related metadata from structured sources
retrieve relevant biological entities across datasets
explain why a gene or pathway is relevant to a disease
clearly distinguish evidence vs inference
avoid overclaiming biological or therapeutic conclusions

In other words, the assistant helps researchers navigate biological knowledge faster while staying explicit about uncertainty.

## MVP User Scenario

The first user scenario focuses on a biomedical researcher exploring disease mechanisms in lung cancer.

Example question:

```text
Which genes are driving NSCLC and what pathways are involved?
```

The assistant should not simply return a list of genes. It should provide structured, evidence-aware reasoning:

```text
EGFR appears as a high-confidence associated gene in NSCLC based on target–disease association scores.

It is involved in the EGFR signaling pathway, which is known to regulate cell proliferation and survival.

However, pathway involvement is inferred from pathway annotation sources and should be treated as mechanistic context rather than direct experimental proof for every dataset entry.
```

This distinction is central to the project: the assistant supports mechanism discovery, not biomedical authority.

## Data Source Strategy

The first version focuses on a single high-quality structured source to reduce complexity and ensure clean reasoning behavior.

### Open Targets

Open Targets provides curated evidence linking genes, proteins, and diseases.

For this project, it is useful for:

- identifying disease-associated genes
- scoring target–disease relationships
- providing evidence types (genetic, clinical, literature, etc.)
- supporting ranked target discovery

However, it does not directly provide full mechanistic explanations, so interpretation must be generated by the assistant carefully.

## Evidence-Aware Matching

The assistant must not treat all gene–disease associations as equal. Instead, it should classify evidence strength.

Example:

```text
Query: Which genes are important in NSCLC?
```

Possible reasoning levels:

- `strong evidence`: high-confidence association score supported by multiple evidence types.
- `medium evidence`: statistically supported but limited mechanistic explanation.
- `weak evidence`: indirect or low-confidence association requiring further validation.

This behavior is important because many dataset discovery questions involve incomplete metadata. The assistant should return useful candidates while labeling uncertainty.

## Project Principles

- Normalize biomedical data into a unified schema (genes, diseases, pathways, targets)
- Preserve evidence metadata from the source rather than flattening it
- Separate retrieval from reasoning logic (RAG layer is replaceable)
- Always distinguish association vs causation
- Avoid clinical or drug-related recommendations
- Prioritize interpretability over complexity

## Example Questions

The assistant is designed to answer research-style questions such as:

- Which genes drive NSCLC?
- What pathways are involved in EGFR-positive lung cancer?
- Which targets appear most promising based on current evidence?

## Does NOT Cover

To maintain scientific clarity and avoid misuse:

- Drug recommendations or prescriptions
- Clinical treatment guidance
- Patient-specific medical advice
- Direct therapeutic decision-making

## Key Question

```text
What biological targets are important, and what evidence supports them?
```

This project focuses on evidence-based target discovery and mechanism interpretation, not medical decision support.
